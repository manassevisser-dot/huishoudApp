### 📄 Bestand: `src/components/fields/MoneyField.tsx`

```tsx
import * as React from 'react';
import MoneyInput from '@ui/components/MoneyInput';
import { useFormContext } from '../../context/FormContext';

type MoneyFieldProps = {
  pageId: string;
  fieldId: string;
  placeholder?: string;
  disabled?: boolean;
  accessibilityLabel?: string;
};

const MoneyField: React.FC<MoneyFieldProps> = ({
  pageId,
  fieldId,
  placeholder = '',
  disabled = false,
  accessibilityLabel = 'Bedrag',
}) => {
  const { state, dispatch } = useFormContext();
  const cents = Number(state?.[pageId]?.[fieldId] ?? 0);

  const handleChange = (nextCents: number) => {
    dispatch({ type: 'SET_PAGE_DATA', pageId, data: { [fieldId]: nextCents } });
  };

  return (
    <MoneyInput
      value={cents}
      onValueChange={handleChange}
      placeholder={placeholder}
      disabled={disabled}
      accessibilityLabel={accessibilityLabel}
    />
  );
};

export default MoneyField;
```

# 📈 Audit-Rapport: WAI-006D (Global Input Hardening)

Als **Lyra (Migration & Test Lead)** heb ik de Change Units voor de "Global Input Swap" geauditeerd. Dit plan is de architecturale voltooiing van de financiële migratie. Door de invoer te centraliseren in `MoneyField`, maken we de UI immuun voor parsing-fouten.

---

## 1️⃣ Strategische Audit per Change Unit

### ✅ WAI-006D-F: De `MoneyField` Dispatcher
Dit is een uitstekende implementatie van het **Dumb Dispatcher** patroon. 
- **Kracht:** Het ontkoppelt de `MoneyInput` (atomaire UI) van de `FormContext` (state). 
- **ADR-15/16:** De component garandeert dat er uitsluitend integers (centen) naar de reducer worden gestuurd. De `Number()` cast bij het inlezen van de state is een noodzakelijke defensieve maatregel voor legacy-data.

### ✅ WAI-006D-I & E: De Repeater Swaps (C7 & C10)
De transitie in de `IncomeRepeater` en `ExpenseRepeater` is de "fysieke sanering". 
- **Cruciaal:** Het verwijderen van `parseFloat` en lokale regex-logica in deze bestanden is de belangrijkste stap om de **"Logic Leaks"** te dichten.
- **Aandachtspunt:** De CU-agent merkt terecht op dat de `fieldId` mapping (bijv. `items.${index}.amount`) afhankelijk is van de kracht van de `formReducer`. Indien de reducer geen "dot-paths" ondersteunt, moet de `UPDATE_MEMBER_FIELD` logica uit WAI-002/003 worden uitgebreid naar een generieke `UPDATE_NESTED_FIELD` actie.

### ✅ WAI-006D-L: De Legacy Cleanup
Dit is de "Search & Destroy" missie. 
- **Audit-eis:** De CU-agent moet na deze stap een `grep -r "parseFloat" src/ui` uitvoeren. Elke hit in de UI-laag die met bedragen te maken heeft, is een stop-conditie.
