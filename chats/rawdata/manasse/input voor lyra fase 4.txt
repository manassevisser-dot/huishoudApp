Fase	Rol van n8n	
FASE 2	Datamodel Vertaling (Nog niet ideaal): Je zult handmatig een datamodel moeten definiëren in n8n om de VBA-structuur te vertalen.	
FASE 3	Migratie-Engine (Zeer geschikt): n8n kan worden gebruikt om de bestaande data uit jouw Excel-database te lezen en automatisch in de nieuwe database (Firebase/Supabase) te schrijven via een workflow.	
FASE 4+	Backend API & Webhooks (Perfect): Je kunt n8n gebruiken om 'Webhooks' op te zetten. Wanneer je mobiele app data wil opslaan of ophalen, stuurt deze een bericht naar n8n, die de logica uitvoert en met de database communiceert. Dit maakt jouw mobiele app heel eenvoudig en verplaatst de complexiteit naar de n8n workflow.	
		
Collectie/Tabel (Firebase/Supabase)	Doel	Essentiële Velden (Fields)
transactions	De ruwe log van alle inkomsten en uitgaven. Dit is de kern van de data.	id (Unieke ID), date (Datum), amount (Bedrag), category (Categorie), description (Omschrijving), type ('Inkomst' of 'Uitgave').
weekly_summaries	Overzichten per ISO-week. Deze worden door de backend (n8n) berekend.	id (Unieke ID), year (Jaar), week_iso (Weeknummer), total_spent (Totaal uitgaven), total_income (Totaal inkomsten).
monthly_summaries	Overzichten per maand. Ook berekend door de backend.	id (Unieke ID), year_month (bijv. "2025-11"), total_spent, total_income.
		
Collectie/Tabel (Supabase)	Doel	Type Relatie
transactions	De ruwe log van alle inkomsten en uitgaven.	Velen
household_settings	Nieuw: Globale instellingen, doelen en statistische input voor de financiële planning.	Eén-op-één (Singelton)
recurring_finances	Nieuw: Inkomen en vaste, maandelijks terugkerende lasten.	Velen
weekly_summaries	Berekende overzichten per week (voor snelle rapportage).	Velen
monthly_summaries	Berekende overzichten per maand.	Velen
		
Logica/Berekening	Vereiste Data	Workflow in n8n
Financiële Remweg	household_settings (Target) + recurring_finances (Netto Maandlasten) + transactions (Huidige Spaartegoed/Buffer).	N8N Workflow leest alle drie de tabellen, voert de berekening uit en slaat het resultaat op in een financial_reports-tabel.
Budget Advies	household_settings (Budget Goal) + monthly_summaries (Huidig Verbruik).	N8N Workflow vergelijkt de monthly_summaries (uitgaven) met de monthly_budget_goal om te bepalen of de gebruiker op schema ligt.

description	text (Text)	Geen	Not Null	De omschrijving van de uitgave.
type	text (Text)	Geen	Not Null	Inkomst' of 'Uitgave'.
