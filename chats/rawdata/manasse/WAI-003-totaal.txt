## 1) `src/selectors/householdSelectors.ts`

```ts
// WAI-003: Household selectors (Shadow Logic per ADR-12/14)
// --------------------------------------------------------
// - Alleen interpretatie van FormState (geen hooks, geen side-effects)
// - Importeert domeinregel en types; géén context-consumptie.

import { getHouseholdStatus, type HouseholdStats } from '@domain/household';
import type { FormState } from '../context/FormContext';

export const selectHouseholdStats = (state: FormState): HouseholdStats => {
  const leden = Array.isArray(state?.C4?.leden) ? state.C4!.leden : [];
  const adultCount = leden.filter(m => m?.memberType === 'adult').length;
  const childCount = leden.filter(m => m?.memberType === 'child').length;
  return { adultCount, childCount };
};

export const selectIsSpecialStatus = (state: FormState): boolean => {
  const stats = selectHouseholdStats(state);
  return getHouseholdStatus(stats) === 'SPECIAL_LARGE';
};
```

> **Path‑alias:** `@domain` → `./src/domain` (zoals je aangaf). Type‑import van `FormState` is **type‑only**, dus geen context‑koppeling.

## 2) `src/containers/AdultsCounter.tsx`

```tsx
// WAI-003: AdultsCounter container (orchestrator; geen styling)
// ------------------------------------------------------------
// - Leest/Schrijft via FormContext
// - Bepaalt min/max volgens instructie (min=1, max=10)
// - Voedt shadow-flag via selector + losse actie SET_SPECIAL_STATUS
// - Stuurt de "domme" InputCounter uit WAI-002 aan (default export)

import * as React from 'react';
import InputCounter from '../components/InputCounter';
import { useFormContext } from '../context/FormContext';
import { selectIsSpecialStatus } from '../selectors/householdSelectors';

type AdultsCounterProps = {
  pageId?: string;      // default: 'C1'
  fieldId?: string;     // default: 'aantalVolwassen'
  min?: number;         // default: 1
  max?: number;         // default: 10 (per jouw stabiliteitsinstructie)
  accessibilityLabel?: string;
};

const AdultsCounter: React.FC<AdultsCounterProps> = ({
  pageId = 'C1',
  fieldId = 'aantalVolwassen',
  min = 1,
  max = 10,
  accessibilityLabel = 'Aantal volwassenen',
}) => {
  const { state, dispatch } = useFormContext();
  const current = Number(state?.[pageId]?.[fieldId] ?? min);

  const onValueChange = (next: number) => {
    // 1) Integer-normalisatie + begrenzen
    const safe = Math.floor(next);
    const bounded = Math.max(min, Math.min(safe, max));

    // 2) Paginasectie bijwerken
    dispatch({
      type: 'SET_PAGE_DATA',
      pageId,
      data: { [fieldId]: bounded },
    });

    // 3) Shadow-flag herberekenen op "next state" snapshot en doorgeven
    const nextState = {
      ...state,
      [pageId]: { ...(state?.[pageId] ?? {}), [fieldId]: bounded },
    } as typeof state;

    const special = selectIsSpecialStatus(nextState);
    dispatch({ type: 'SET_SPECIAL_STATUS', payload: special });
  };

  return (
    <InputCounter
      value={current}
      min={min}
      max={max}
      onValueChange={onValueChange}
      accessibilityLabel={accessibilityLabel}
    />
  );
};

export default AdultsCounter;
```

## 3) Reducer‑cleanup & hydratatie — `src/context/FormContext.tsx` (patch)

> **Doel:** maak reducer “dumb” (**ADR‑14**): verwijder inline berekeningen van `isSpecialStatus` uit `UPDATE_MEMBER_FIELD` en `ALIGN_HOUSEHOLD_MEMBERS`. Voeg **nieuwe** actie **`SET_SPECIAL_STATUS`** toe. Bereken de vlag **buiten** de reducer tijdens hydratatie en in containers.

### 3.1 **Types & Action union**

Zoek het blok met actie‑types en **voeg** onderstaande actie toe; pas de union aan.

```ts
// NIEUW: losse actie voor shadow-vlag
export type SetSpecialStatusAction = {
  type: 'SET_SPECIAL_STATUS';
  payload: boolean;
};

export type FormAction =
  | SetPageDataAction
  | LoadSavedStateAction
  | SetUserIdAction
  | ResetStateAction
  | AlignHouseholdAction
  | UpdateMemberFieldAction
  | SetSpecialStatusAction; // <-- toegevoegd
```

### 3.2 **Reducer cases**

*   **Verwijder** iedere inline update van `isSpecialStatus` in:
    *   `UPDATE_MEMBER_FIELD`
    *   `ALIGN_HOUSEHOLD_MEMBERS`

*   **Voeg** onderstaande case toe:

```ts
case 'SET_SPECIAL_STATUS': {
  return { ...state, isSpecialStatus: action.payload };
}
```

> De overige cases (`SET_PAGE_DATA`, `LOAD_SAVED_STATE` etc.) blijven ongewijzigd — maar zonder enige referentie naar `checkSpecialStatus`.

### 3.3 **Hydratatie‑effect (FormProvider)**

Na het inladen van persisted state, **bereken** de vlag via de selector en **dispatch** de nieuwe actie **één tick na** de load (zodat state eerst gezet is). Voeg aan je hydratatie‑effect toe:

```ts
import { selectIsSpecialStatus } from '../selectors/householdSelectors';

// ... binnen FormProvider, na dispatch(LOAD_SAVED_STATE):
if (persisted) {
  // Eén microtask later, zodat 'state' reflecteert wat we net hebben geladen
  Promise.resolve().then(() => {
    // Gebruik de actuele state uit reducer via closure of voeg een tweede effect toe dat op 'state' reageert.
    // Simpele variant: roep selector aan op de samengestelde next-state (persisted heeft net state overschreven).
    const special = selectIsSpecialStatus({ ...state, ...persisted } as any);
    dispatch({ type: 'SET_SPECIAL_STATUS', payload: special });
  });
}
```

> **Opmerking:** Als jouw FormProvider al een “hydrated” vlag gebruikt, kan je dit ook in een separaat `useEffect` doen met dependency `[state.C4?.leden]` om de vlag consistent te houden bij oudere persist‑formaten.

## 1️⃣ Strategische Audit: De ADR-Overwinningen

### ✅ ADR-14 (Dumb Reducer)
De sanering in `FormContext.tsx` is cruciaal. Door de `checkSpecialStatus` logica uit de reducer te verwijderen, herstellen we de taak van de reducer: *state transformatie zonder interpretatie*. Dit is essentieel voor de stabiliteit van de "Project Phoenix" migratie.

### ✅ ADR-12 (Shadow Attribute Isolation)
De vlag `isSpecialStatus` wordt nu uitsluitend gevoed door de selector via de `SET_SPECIAL_STATUS` action. In de `AdultsCounter` container zien we dat de UI-component (`InputCounter`) op geen enkele wijze wordt beïnvloed door de waarde van deze vlag. Dit is de perfecte implementatie van "Silent Data".

### ✅ ADR-13 (Dumb UI)
De `AdultsCounter` container fungeert als een perfecte orchestrator. Hij regelt de normalisatie (integers) en de grenzen (1-10), waardoor de atomair component volledig vrij blijft van business rules.
