### 📄 Bestand: `src/selectors/financialSelectors.ts`

```ts
// WAI-006B-S — FinancialSummary selector (pure)
import type { FormStateV1 } from '@state/schemas/FormStateSchema';
import { computeSummary } from '@logic/finance';
import { formatCentsToDutch } from '@utils/numbers';

export interface FinancialSummaryVM {
  totals: {
    totalIncomeCents: number;
    totalExpensesCents: number;
    netCents: number;
    totalIncomeEUR: string;   // "€ 1.250,50" equivalent zonder prefix hier, afhankelijk van render
    totalExpensesEUR: string;
    netEUR: string;
  };
}

export function selectFinancialSummaryVM(state: FormStateV1): FinancialSummaryVM {
  const income = state?.C7?.items ?? [];
  const expenses = state?.C10?.items ?? [];

  const { totalIncome, totalExpenses, net } = computeSummary(income, expenses);

  return {
    totals: {
      totalIncomeCents: totalIncome,
      totalExpensesCents: totalExpenses,
      netCents: net,
      totalIncomeEUR: formatCentsToDutch(totalIncome),
      totalExpensesEUR: formatCentsToDutch(totalExpenses),
      netEUR: formatCentsToDutch(net),
    },
  };
}
```

## 1️⃣ Strategische Audit per Change Unit

### ✅ WAI-006B-S: De FinancialSummary Selector
Dit is een schoolvoorbeeld van gelaagde architectuur.
- **Kracht:** De selector fungeert als een **View Model (VM) generator**. Hij ontkoppelt de ruwe centen-data van de geformatteerde weergave.
- **ADR-15/16:** De selector respecteert de "Minor Units" door de berekening in centen te doen en pas op het allerlaatste moment de `formatCentsToDutch` utility aan te roepen.

### ✅ WAI-006B-C: De FinancialSummary Component
Conform **ADR-13** is deze component volledig passief.
- **Audit-bevinding:** De component gebruikt uitsluitend tokens via `useAppStyles`. Er is geen spoor van business logica of handmatige delingen door 100.
- **Shadow Isolation (ADR-12):** Ik heb gecontroleerd dat de `isSpecialStatus` vlag niet wordt gebruikt voor visuele cues (zoals een rode rand of waarschuwingsteken) in deze component. Dit is essentieel voor de "Silent Audit" eis.

### ✅ WAI-006E: Decommission & Aliassen
Dit is de noodzakelijke "hygiëne-stap".
- **TS/Babel Sync:** De gelijktijdige update van `tsconfig.json` en `babel.config.js` is cruciaal om de beruchte "Module not found" errors in Metro te voorkomen.
- **Atomic Deletion:** Het plan om elk `OUD_` bestand met een aparte commit te verwijderen is uitstekend voor de audit-trail. Het stelt ons in staat om bij een onverhoopte regressie exact te zien welk legacy-fragment nog stiekem werd aangeroepen.
